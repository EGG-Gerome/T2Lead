# Architecture Note

## Required sections

1. Problem statement
2. Scope and non-goals
3. Candidate options and trade-offs
4. Chosen design and why
5. Risks, rollback plan, and test plan

## Output style

- Keep it short, decision-oriented, and reproducible.
- Include concrete file paths and config keys when relevant.
