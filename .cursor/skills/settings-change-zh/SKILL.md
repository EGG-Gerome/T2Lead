# 配置修改流程（中文）

## 步骤

1. 明确影响范围（单次运行还是全局默认）。
2. 修改配置源（YAML / DP_* / CLI）并说明优先级。
3. 补充对应文档（配置参考、README 示例）。
4. 运行最小验证命令确认配置生效。
5. 记录 `git diff` 便于审阅。

## 必须输出

- 变更的配置键路径
- 生效方式（文件/环境变量/CLI）
- 验证证据（日志或命令输出）
