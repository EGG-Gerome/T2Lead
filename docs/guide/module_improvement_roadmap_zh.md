# 模块改进路线图（精度 / 速度 / 可扩展）

本文用于任务拆分与新人 onboarding，聚焦“当前可跑通”与“后续可提升”。

## A. 变异输入与标准化

### 当前
- 支持 Xena TSV、VEP 注释 VCF、FASTQ（经 Sarek 上游）。

### 可改进
- 增加更强的输入质量校验（样本列、参考基因组版本、注释完整度）。
- 增加批处理分片器（按样本/按基因/按变异数）与失败自动重试。

## B. 结构解析与突变建模

### 当前
- 实验 PDB 优先；
- 点突变 + 局部约束优化；
- 失败回退 ESMFold；
- 再失败尝试 AlphaFold DB。

### 可改进
- 引入本地折叠后端（本地 ESMFold / ColabFold-AF2）减少外部 API 依赖。
- 针对超长蛋白增加“域分块+拼接”策略。
- 引入结构质量评分（pLDDT、模板覆盖率、局部几何检查）作为后续筛选权重。

### 推荐落地策略（开源优先）
1. **快速筛选层**：先用实验 PDB + ESMFold（快、吞吐高）；
2. **精修层**：对 top 候选位点再用 ColabFold-AF2 做更高质量建模；
3. 对超长序列走分块/域级建模，避免单次 OOM。

### 关于 AF3
- AlphaFold3 当前并非完整开源可本地自由部署的传统模式（更多是服务/API 形态与限制）。
- 生产可控角度建议优先开源可落地链路：实验 PDB + AFDB + 本地 ESMFold/AF2。

## C. 对接与 MD（阶段四）

### 当前
- Vina + ADMET 硬过滤 + MM-GBSA 风格近似评分 + RMSD 稳定性。

### 可改进
- 自动选择更优对接盒与受体预处理参数。
- 对关键候选增加更长 MD 与重复次数，提升排序鲁棒性。
- 建立“快速筛选模式 / 严格评估模式”双配置。

## D. 靶点发现策略

### 当前
- 基于突变位点与基因结构路径推进；可自动 Stage2-3。

### 可改进
- 融合病人层面的转录组/代谢组特征，提高靶点优先级个体化程度。
- 增加通路网络层面重加权（驱动基因 + 表达活性 + 可药性）。

## E. RNA 疗法分支（规划）

### 目标
- 当“小分子靶点路径找不到可行候选”时，进入 RNA 疗法候选分支。

### 建议起步
1. 定义触发条件（例如无可药靶点、或阶段四可靠候选低于阈值）。
2. 输出 RNA 分支输入包（目标基因、突变位点、表达信息、优先级）。
3. 与 siRNA/ASO 设计工具链对接，形成可追踪子流程。

## F. 运维与可复现

### 当前
- 已有 run 归档规范与日志路径。

### 可改进
- 增加“统一启动器”（后台运行 + 自动日志 + 自动归档）。
- 增加 run 级 dashboard（每批次进度、成功率、失败原因分布）。

## 优先级建议（近期）

1. 批处理分片 + 自动恢复（提升大规模稳定性）。
2. 本地结构后端（降低 ESMFold API 限制影响）。
3. 快速/严格双模式配置（兼顾吞吐与质量）。

## G. OpenMM 受控升级计划（8.2 -> 8.5.1）

### 背景
- 当前环境实测为 OpenMM `8.2`。
- 上游最新稳定补丁为 `8.5.1`（含多项 bugfix）。

### 升级策略
1. 在独立分支执行升级（不影响主分支运行）。
2. 保持其余依赖尽量不变，先做最小升级。
3. 跑小规模基准对比：
   - 同一受体、同一小分子集合；
   - 比较能量最小化稳定性、MD 成功率、耗时。
4. 若结果一致且无回归，再合并到主流程配置。

### 验收标准
- 关键测试通过；
- 小规模 benchmark 无显著精度/稳定性退化；
- 运行耗时可接受；
- 文档同步更新（版本、变更点、回滚方式）。
