# DrugReflector 模块任务书（姜可盈）

**日期：** 2026-04-18  
**模块定位：** 独立表型药物发现模块（新建独立 GitHub 仓库）  
**目标：** 基于单细胞转录组数据构建疾病表达特征，使用 DrugReflector 进行候选药物排序，产出可复现的分析流程与结果报告。

---

## 1) 向华大基因（BGI）要的数据

1. **建议要的数据**
   - **原始数据**：FASTQ（便于追溯和二次分析）
   - **标准处理结果**：单细胞计数矩阵（`.h5` / `.h5ad` / `.rds`）
   - **细胞与样本元数据**（必须）：样本分组（肿瘤/正常）、患者 ID、批次、组织来源、细胞注释（若有）
2. **“单细胞计数矩阵”是不是单细胞转录组数据？**  
   是核心数据，但**不完整**。缺少元数据时无法可靠做分组比较与下游建模。
3. **要不要让华大直接给 DEGs（LFC+p-value）？**  
   可以让他们给作为参考，但仍**需自己复算 DEA**（保证参数可控和可复现）。
4. **FASTQ -> 矩阵这步要不要你们做？**  
   若华大提供质量可靠的 `.h5ad/.rds` + 完整元数据，这步可不做。  
   但建议保留 FASTQ 以便后续审计/复算。
5. **DrugReflector 输入是不是 DEGs 表？**  
   DrugReflector更偏向输入“表达签名/v-score 向量”，不是仅一个短 DEGs 列表。  
   DEGs（LFC/p-value）是中间结果和解释层，不是唯一输入形态。

---

## 2) 给华大（BGI）的数据需求清单

### A. 必须交付（Must Have）
- 单细胞计数矩阵：`.h5ad`（优先）或 `.rds`（Seurat）或 10x `.h5`。
- 细胞元数据（cell-level metadata）：至少包含  
  `cell_barcode`, `sample_id`, `patient_id`, `condition(肿瘤/正常)`, `batch`, `tissue/source`。
- 基因注释：`gene_id`、`gene_symbol`（HGNC 格式优先）。
- 基础质控结果：每样本测序深度、有效细胞数、线粒体比例分布、双细胞处理说明。

### B. 强烈建议（Strongly Recommended）
- 原始 FASTQ（用于追溯、复跑与审计）。
- 上游流程说明：软件版本、参考基因组、比对/定量参数。
- 若有：初步细胞类型注释与聚类结果（作为参考，不直接当最终结论）。

### C. 可选（Nice to Have）
- 华大提供的 DEA 初稿（含 `log2FC`, `p-value`, `adj_p-value`）。
- 样本临床标签（脱敏后）：分期、治疗史、关键病理特征。

## 3) 工作范围（模块边界）

### AI提供的模块名称参考

```
scRNA-signature-to-drugreflector
```

### In-Scope（要做）
- 从表达矩阵与元数据出发完成：
  - 质量控制（QC）
  - 归一化与批次处理（按数据情况）
  - 细胞分群/细胞状态定义
  - 肿瘤 vs 正常（或目标细胞群 vs 对照群）DEA
  - 疾病签名构建（v-score 或等价签名）
  - DrugReflector 推理与药物排序
  - 结果解释（Top compounds + 关键驱动基因/通路）

---

## 4) 建议执行步骤

1. **数据接收与审计**
   - 检查是否具备矩阵、元数据、分组标签、基因注释。
   - 输出数据审计清单（缺失字段、异常样本）。

2. **预处理与分群**
   - 用 Scanpy 或 Seurat 完成 QC、标准化、降维、聚类。
   - 明确比较对象：癌细胞群 vs 正常对照群（避免混淆细胞类型差异与疾病差异）。

3. **差异表达与签名构建**
   - 产出 DEGs 表：`gene`, `log2FC`, `p-value`, `adj_p-value`。
   - 从全基因结果构建签名（而非只取少量 DEGs）。
   - 做通路富集作为生物学合理性检查（防“分析跑歪”）。

4. **DrugReflector 推理**
   - 准备 DrugReflector 所需输入（注意基因名标准化、覆盖度检查）。
   - 运行模型并输出药物排序结果（Top-N，建议 N=20/50 两档）。
   - 对高分候选给出可解释注释（已知机制、适应症、潜在冲突）。

5. **复现与交付**
   - 一键复现脚本或 notebook pipeline。
   - 输出报告、图表、结果表格和运行说明。

## 5) 最终交付物

### 代码与工程
- 独立 GitHub 仓库（含 README、环境配置、运行命令）。
- 可复现分析脚本（从输入矩阵到药物排序结果）。

### 数据与结果
- `DEA_results.csv`：至少含 `gene`, `log2FC`, `p-value`, `adj_p-value`。
- `disease_signature.csv`：用于 DrugReflector 的签名向量。
- `drugreflector_top_hits.csv`：药物排名结果（建议含分数、rank、基础注释）。
- 关键图件：UMAP/聚类图、火山图、通路富集图、药物排名图。（可选）

### 报告
- `REPORT.md`（5-10 页等价内容）：
  - 数据来源和样本概况
  - 方法与参数
  - 核心结果
  - 风险与局限
  - 下一步实验建议

## 6) 验收标准（通过/不通过）

- **可复现性**：新环境按 README 可完整跑通。
- **完整性**：DEA + 签名 + DrugReflector 结果三件套齐全。
- **可解释性**：Top 候选药物有机制/文献级解释，不是纯黑盒分数。
- **质量控制**：报告中明确展示 QC 与批次影响处理思路。
- **边界清晰**：明确声明该模块不等同于新抗原疫苗设计模块。
