# T2Lead 项目进展与规划汇报

> 2026-04-01 · 阶段性总结

---

## 一、项目定位

T2Lead 的核心目标是建立**从患者 DNA 外显子到个性化候选小分子配体**的全自动闭环——输入患者肿瘤组织与正常血液样本的外显子测序数据，自动完成体细胞突变检测、突变蛋白结构预测、候选分子生成与多维度评估，最终输出针对该患者突变靶点的候选药物分子排序，并与该靶点已有优秀药物进行可视化对比。后续将进一步接入患者代谢组学等多维数据，实现更精准的个体化评分。

在此基础上，项目同时实现了"输入疾病名称 → 自动完成靶点发现到先导优化"的通用药物发现流水线，作为核心能力的验证平台和技术底座。

核心设计原则：**单命令全自动、全链路可复现、每个环节可独立替换**。

**工程能力指标（乳腺癌场景实测）：**

| 指标 | 数值 |
|------|------|
| 首次完整运行（冷启动：含 ChEMBL 数据下载、285 万 Morgan 指纹首次计算，以及训练、全库打分与属性、Stage 3/4、10 分子 MM-GBSA） | ~24 小时（GPU） |
| 再次运行（ChEMBL 与 285 万指纹已缓存，**不含**上述下载与指纹首次计算；仍重跑训练、285 万预测与属性、对接、MM-GBSA） | 约 **8–9 小时**（GPU）；其中 **MM-GBSA 约 7.5–8 小时**（10 个候选；实测日志 `20260401_004727`：全流程 00:47→09:50，MD 段约 02:15→09:50） |
| 虚拟筛选库规模 | 285 万分子 |
| REINVENT4 RL 新生成分子 | 12,386 个有效唯一结构 |
| 模型预测精度 | RF R²=0.77, CV R²=0.73±0.02 |
| 对接成功率 | 50/50 (100%) |
| ADMET 通过率 | 10/10 全部通过 hERG/CYP/Veber |

**耗时说明：** 日志里「ChEMBL crawl skipping」「Loaded cached fingerprints」只表示**抓取步骤与 Morgan `.npy` 可跳过或秒级加载**，不是整条流水线重跑只需秒级。当前实现下，**模型训练、285 万分子 pIC50 预测与物性计算、Vina 对接、MM-GBSA 每次运行都会重新执行**，对接与 MD **没有**结果级缓存；若仅改配置做小范围调试，可依赖阶段产物手动断点续跑，但默认全量重跑时间与上表一致。

**创新点 ①：智能靶点选择策略**

不盲选关联评分最高的靶点——BRCA1 虽然遗传学证据评分最高（0.909），但 ChEMBL 仅 20 条 IC50 数据，无法训练可靠模型。系统自动切换到数据充足的 PIK3CA（9,728 条），保证了下游 ML 模型的统计效力。这一逻辑在其他公开流水线中未见。

**创新点 ②：RL 生成 × 传统筛选的融合架构**

Stage 3 同时运行 CREM 骨架改造（传统化学空间搜索）和 REINVENT4 强化学习（目标导向生成），两条路径的分子回流后统一 MPO 排序，兼顾化学空间覆盖度和目标导向性。

**创新点 ③：权重自适应 composite scoring**

opt_score 的各维度权重在组件缺失时自动重归一化（如 MD 失败则 docking + ADMET 自动接管全部权重），保证流水线在任何硬件配置下都能给出有意义的排序，而非整体失败。

## 三、正在推进：外显子到小分子配体的闭环（项目核心）

### 目标

实现"患者肿瘤组织外显子 + 正常血液外显子 → 体细胞突变检测 → 突变蛋白结构预测 → 针对性小分子配体生成与评估"的全自动闭环。

### 架构

```
 肿瘤组织外显子 FASTQ + 正常血液外显子 FASTQ
                    │
                    ▼
          ┌─────────────────────┐
          │  nf-core/sarek       │  BWA-MEM 比对 → Mutect2 体细胞突变检测
          │  (Nextflow 容器化)   │  → VEP 变异注释
          └─────────┬───────────┘
                    │ 注释后 VCF
                    ▼
          ┌─────────────────────┐
          │  VCF Parser          │  提取蛋白层变异：EGFR L858R, PIK3CA H1047R ...
          │  (支持 VEP/SnpEff)  │  过滤 MODERATE + HIGH impact
          └─────────┬───────────┘
                    │ SomaticVariant 列表
                    ▼
          ┌─────────────────────┐
          │  Mutant Sequence     │  UniProt/Ensembl 获取野生型序列
          │  Builder             │  → 应用氨基酸替换 → 输出突变 FASTA
          └─────────┬───────────┘
                    │ 突变蛋白序列
                    ▼
          ┌─────────────────────┐
          │  Structure Bridge    │  决策树：
          │                      │  ① RCSB 有实验结构？→ 下载 + 点突变
          │                      │  ② 无 → ESMFold/Boltz-2 预测突变结构
          └─────────┬───────────┘
                    │ 突变蛋白 PDB
                    ▼
          ┌─────────────────────┐
          │  Stage 4 (复用)      │  对每个突变体独立运行：
          │  对接 + MD + 排序    │  docking → ADMET → MD → opt_score
          │                      │  输出路径: stage4_optimization/<GENE_MUT>/
          └─────────┬───────────┘
                    │
                    ▼
          ┌─────────────────────┐
          │  Benchmark 自动对比  │  自动查询该靶点已有最佳药物
          │  + 可视化报告        │  → head-to-head 评分对比 → HTML 报告
          └──────────────────────┘
```

### 延伸思路：健康基线样本存档

除肿瘤/正常血液配对外，一个值得探索的方向是**在个体健康时期存储基线细胞或组织样本**。相较于癌变后取同一患者的血液作为"正常对照"，健康时期的同源组织样本能提供更纯净的胚系基线——血液样本虽然是标准做法，但在血液系统受肿瘤微环境影响（如 cfDNA 污染、克隆性造血）的情况下，预存的健康基线理论上能进一步提高体细胞突变检测的特异性。这可以作为未来数据采集策略的一个方向。

| VCF Parser | 代码完成 | 内置轻量解析，可选 pysam/cyvcf2 加速 |

| 端到端验证 | 待进行 | 需要真实肿瘤/正常 FASTQ 数据做首次全链路测试 |

**创新点 ④：从基因型到候选药物的全自动桥接**

目前绝大多数药物发现流水线止步于"给定靶点 → 筛选分子"。T2Lead 将上游的体细胞突变检测与下游的结构预测 + 对接评估打通：一份外显子数据进去，出来的是"这个患者的突变蛋白，哪些候选分子结合最好"的排序表。这在同类开源工具中是独有的。

**创新点 ⑤：结构预测策略自动路由**

Structure Bridge 实现了多级后备策略：实验结构 → ESMFold API → 本地 ESMFold → ColabFold，自动选最优可用路径。后续 Boltz-2 / Chai-1 作为可插拔后端接入。

## 四、项目规划

### 4.1 自动化 Benchmark 对比

流水线确定靶点后（如 PIK3CA），自动从 ChEMBL 查询该靶点已批准或临床阶段最高的药物分子（如 PIK3CA → Alpelisib），将这些参考药物送入同一套评分体系（pIC50 预测、QED、MPO、对接、ADMET、opt_score），生成与我们候选分子的 head-to-head 对比。让每次运行的输出自带"和最好的已有药比，我们的分子好在哪、差在哪"的完整回答。

### 4.2 可视化报告

在 Stage 4 结束后自动生成交互式 HTML 报告（浏览器打开即可，无需服务器），包含：总览卡片、候选分子排名表（交互式排序筛选）、pIC50 vs Docking 气泡图、Top N 雷达图多维对比、ADMET 通过热图、2D 分子结构渲染、以及 Benchmark 对比结果。替代当前纯 CSV + 日志的输出方式。

### 4.3 患者个体化评分加权

将患者自身的生物学特征转化为评分权重调节因子，使同一批候选分子对不同患者产生不同的优先级排序。规划三个实施层次：

**层次 1 · 代谢酶分型驱动（推荐起步）**

| 维度 | 说明 |
|------|------|
| 输入 | 患者 CYP 基因分型（CYP2D6 慢代谢/中间型/正常/超快代谢） |
| 思路 | 查表映射为代谢风险系数 → 动态调整 opt_score 中 CYP 相关权重 |
| 输出 | 个性化 opt_score，不同代谢类型的患者看到不同排序 |
| 难点 | 低，临床 PGx 检测已标准化，PharmGKB 有完整映射表 |

**层次 2 · 多组学小模型加权**

| 维度 | 说明 |
|------|------|
| 输入 | 舌苔代谢组（LC-MS）、肠道菌群（16S）、血液转录组（RNA-seq）等 |
| 思路 | 每个数据源训练独立轻量模型，输出 0-1 分数，映射到不同评分维度的权重调节 |
| 输出 | 多维度个性化 opt_score |
| 难点 | 各组学数据的标准化采集方案；每个模型需要足够的训练验证队列；权重映射需临床可解释性 |

**层次 3 · 端到端个性化药物响应预测**

| 维度 | 说明 |
|------|------|
| 输入 | 患者完整多组学特征 + 候选分子指纹 |
| 思路 | 端到端深度学习（类似 DeepCDR / TGSA），患者编码器 × 分子编码器 → 直接预测响应 |
| 输出 | 该患者对该分子的响应预测，作为最终排序依据 |
| 难点 | 需大规模患者-药物响应配对数据（公开数据多为细胞系非真实患者）；可解释性不足 |

**推荐路径：** 先落层次 1（改动小但概念完整），架构上预留层次 2 接口（瓶颈在数据而非代码），层次 3 作为中长期研究方向。

## 五、技术高地与核心竞争力

| # | 能力 | 说明 | 同类工具对比 |
|---|------|------|------------|
| 1 | **外显子到候选配体一键闭环** | 肿瘤/正常外显子 → 突变检测 → 结构预测 → 分子生成/评估 | 同类开源工具中未见 |
| 2 | **自动化 Benchmark 对比** | 自动查询靶点已有最佳药物并 head-to-head 评分（规划中） | 需手动收集参考数据 |
| 3 | **RL 生成 + 传统筛选融合** | REINVENT4 与 CREM 双路径产出统一排序 | 通常只选其一 |
| 4 | **物理验证层（非纯 ML）** | AutoDock Vina + OpenMM MD + MM-GBSA | 多数止步于 ML 预测 |
| 5 | **容错自愈架构** | MD 失败→权重重分配；REINVENT 失败→fallback；靶点数据不足→自动切换 | 通常整段失败 |
| 6 | **三级缓存系统** | 指纹/模型/筛选结果全缓存，迭代开发 <10 秒重跑 | 多数每次从头计算 |
| 7 | **多入口场景支持** | 外显子模式 / 疾病名模式 / 外部数据 / 纯对接 / 蛋白序列模式 | 通常只有单一入口 |
